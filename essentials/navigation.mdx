Roundproxies residential proxies provide access to a global network of residential IP addresses with advanced targeting capabilities. Our proxies support country, state, and city-level targeting, along with session management for consistent IP addresses.

### Proxy Structure

Our residential proxies follow a standardized format that allows precise geographic targeting and session control:

`residential.roundproxies.com:5000:client-{CLIENT_ID}-{TARGETING_PARAMS}-session-{SESSION_ID}:{PASSWORD}`

### Components Breakdown

| Component | Description | Example |

|-----------|-------------|---------|

| **Host** | Proxy server hostname | `residential.roundproxies.com` |

| **Port** | Connection port | `5000` |

| **Client ID** | Your unique client identifier | `client-77ec791562d243759695e8fb5dc07a81` |

| **Targeting** | Geographic targeting parameters | `country-Israel-state-northerndistrict-city-maghar` |

| **Session** | Session identifier for IP persistence | `session-lw515ejc` |

| **Password** | Authentication password | `df1a09d0542141d1b910d5d7e7a5d5ab` |

## Geographic Targeting

### Country Targeting

Target specific countries by including the country parameter in your proxy credentials:

\`\`\`

country-{COUNTRY_NAME}

\`\`\`

**Available countries in examples:**

- Israel

- Hungary

- Germany

### State/Region Targeting

Narrow down to specific states or regions:

\`\`\`

state-{STATE_NAME}

\`\`\`

**Examples:**

- `state-northerndistrict` (Israel)

- `state-pestcounty` (Hungary)

- `state-citystatebremen` (Germany)

- `state-landberlin` (Germany)

### City Targeting

Target specific cities for maximum precision:

\`\`\`

city-{CITY_NAME}

\`\`\`

**Examples:**

- `city-maghar` (Israel)

- `city-god` (Hungary)

- `city-bremen` (Germany)

- `city-berlin` (Germany)

## Session Management

Sessions allow you to maintain the same IP address across multiple requests, which is essential for activities requiring consistent identity.

### Session Format

\`\`\`

session-{SESSION_ID}

\`\`\`

The session ID can be any alphanumeric string. Using the same session ID will route your requests through the same IP address.

**Example session IDs:**

- `session-lw515ejc`

- `session-gbyii4th`

- `session-wwmchxfs`

- `session-fnzwtath`

\<Tip\>

Generate unique session IDs for different browsing sessions or tasks to maintain separate IP identities.

\</Tip\>

## Best Practices

\<AccordionGroup\>

  \<Accordion title="Session Management"\>

    - Use unique session IDs for different tasks or browsing sessions

    - Keep session IDs consistent when you need to maintain the same IP

    - Rotate session IDs periodically to distribute load

  \</Accordion\>

  \<Accordion title="Geographic Targeting"\>

    - Use the most specific targeting available for your use case

    - Consider timezone and language when selecting locations

    - Test different locations to find optimal performance

  \</Accordion\>

  \<Accordion title="Performance Optimization"\>

    - Reuse connections when possible to reduce latency

    - Implement proper error handling and retry logic

    - Monitor response times and adjust targeting as needed

  \</Accordion\>

  \<Accordion title="Security"\>

    - Never expose your password in client-side code

    - Store credentials securely using environment variables

    - Rotate passwords regularly for enhanced security

  \</Accordion\>

\</AccordionGroup\>

\<Warning\>

Keep your client ID and password confidential. Anyone with these credentials can use your proxy allocation.

\</Warning\>

## Troubleshooting

\<AccordionGroup\>

  \<Accordion title="Connection Refused"\>

    - Verify the hostname and port are correct

    - Check if your IP is whitelisted (if applicable)

    - Ensure your account has sufficient proxy credits

  \</Accordion\>

  \<Accordion title="Authentication Failed"\>

    - Double-check your client ID and password

    - Ensure there are no extra spaces in the username

    - Verify your account is active

  \</Accordion\>

  \<Accordion title="Wrong Geographic Location"\>

    - Verify the country, state, and city names are spelled correctly

    - Use lowercase with no spaces (use hyphens for multi-word names)

    - Check if the requested location is available in your plan

  \</Accordion\>

  \<Accordion title="IP Changes During Session"\>

    - Ensure you're using the same session ID

    - Check if the session has expired (session duration varies)

    - Verify network connectivity hasn't been interrupted

  \</Accordion\>

\</AccordionGroup\>
